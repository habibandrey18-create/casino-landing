Ниже описаны самые дешевые варианты, начиная с БЕСПЛАТНЫХ.

═══════════════════════════════════════════════════════════════════════════════
ВАРИАНТ 1: RENDER.COM (БЕСПЛАТНО) ⭐ РЕКОМЕНДУЮ
═══════════════════════════════════════════════════════════════════════════════

Render.com - один из лучших бесплатных вариантов для Node.js.

ПЛЮСЫ:
✅ Полностью бесплатно
✅ Поддержка Node.js
✅ Автоматическое развертывание из GitHub
✅ Бесплатный SSL сертификат (HTTPS)
✅ Автоматические обновления при изменении кода

МИНУСЫ:
⚠️ После 15 минут бездействия сервер "засыпает" (первый запрос будет медленным)
⚠️ Ограниченные ресурсы (но для вашего сайта достаточно)

ШАГИ:

1. СОЗДАНИЕ АККАУНТА НА GITHUB (если нет):
   - Откройте https://github.com
   - Нажмите "Sign up" (Регистрация)
   - Создайте бесплатный аккаунт
   - Подтвердите email

2. ЗАГРУЗКА ПРОЕКТА НА GITHUB:
   
   а) Установите Git (если не установлен):
      - Скачайте: https://git-scm.com/download/win
      - Установите (всё по умолчанию, нажимайте Next)
   
   б) Откройте командную строку в папке проекта:
      - Перейдите в папку: cd "C:\Users\NewAdmin\Desktop\сайт"
   
   в) Выполните команды:
      
      git init
      git add .
      git commit -m "Initial commit"
      
      (Если Git попросит настроить имя и email, выполните):
      git config --global user.name "Ваше Имя"
      git config --global user.email "ваш@email.com"
      
      Затем снова:
      git commit -m "Initial commit"
   
   г) Создайте репозиторий на GitHub:
      - Зайдите на https://github.com
      - Нажмите зелёную кнопку "New" (или "+" → "New repository")
      - Название: casino-landing (или любое другое)
      - Выберите "Public" (публичный)
      - НЕ ставьте галочки (не создавайте README, .gitignore, лицензию)
      - Нажмите "Create repository"
   
   д) Загрузите код на GitHub:
      (GitHub покажет команды, но вот они):
      
      git remote add origin https://github.com/ВАШ_ЛОГИН/casino-landing.git
      git branch -M main
      git push -u origin main
      
      (Замените ВАШ_ЛОГИН на ваш логин GitHub)
      
      Вас попросят ввести логин и пароль GitHub

3. СОЗДАНИЕ АККАУНТА НА RENDER:
   - Откройте https://render.com
   - Нажмите "Get Started for Free"
   - Войдите через GitHub (нажмите "Continue with GitHub")
   - Разрешите доступ Render к вашему GitHub аккаунту

4. СОЗДАНИЕ НОВОГО СЕРВИСА:
   - Нажмите "New +" → "Web Service"
   - Выберите ваш репозиторий (casino-landing)
   - Настройки:
     * Name: casino-landing (или любое название)
     * Region: Frankfurt (или ближайший к вам)
     * Branch: main
     * Root Directory: (оставьте пустым)
     * Runtime: Node
     * Build Command: npm install
     * Start Command: npm start
     * Plan: Free (бесплатный)
   
   - Нажмите "Create Web Service"
   - Render начнёт развертывание (займёт 2-5 минут)

5. НАСТРОЙКА ПЕРЕМЕННЫХ ОКРУЖЕНИЯ:
   - В Render нажмите на ваш сервис
   - Перейдите в раздел "Environment"
   - Добавьте переменные:
     * ADMIN_PASSWORD = ваш_пароль_админки
     * JWT_SECRET = любой_случайный_текст_для_безопасности
     * PORT = 10000 (Render сам назначит порт, но можно указать)
     * NODE_ENV = production
   - Нажмите "Save Changes"
   - Сервис автоматически перезапустится

6. ПОЛУЧЕНИЕ ССЫЛКИ:
   - После развертывания вы получите ссылку вида:
     https://casino-landing.onrender.com
   - Эта ссылка будет работать всегда
   - SSL (HTTPS) настроится автоматически

ВАРИАНТ 4: VPS СЕРВЕР (ОТ 2-3$ В МЕСЯЦ) - ЕСЛИ БЕСПЛАТНОЕ НЕ ПОДОЙДЕТ
═══════════════════════════════════════════════════════════════════════════════

Если бесплатные варианты не подходят (слишком медленные или ограничения),
можно купить дешевый VPS.

РЕКОМЕНДУЕМЫЕ ПРОВАЙДЕРЫ:

1. HETZNER (Германия) - от 2.96€/месяц
   - Сайт: https://www.hetzner.com
   - VPS с 1 CPU, 2GB RAM, 20GB SSD
   - Очень быстрый и надежный
   - Нужна карта для оплаты

2. DIGITALOCEAN (США) - от 4$/месяц
   - Сайт: https://www.digitalocean.com
   - Хорошая документация
   - Простая панель управления

3. VULTR (Глобально) - от 2.50$/месяц
   - Сайт: https://www.vultr.com
   - Много локаций
   - Хорошая производительность

КАК НАСТРОИТЬ VPS:

1. Купите VPS (выберите Ubuntu 22.04)
2. Получите IP адрес и пароль root
3. Подключитесь через SSH (используйте PuTTY для Windows)
4. Установите Node.js:
   
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt-get install -y nodejs
   
5. Установите PM2 (для автозапуска):
   
   sudo npm install -g pm2
   
6. Загрузите файлы проекта (через SFTP или Git)
7. Создайте файл .env с переменными
8. Запустите: pm2 start server.js --name casino-landing
9. Настройте автозапуск: pm2 startup && pm2 save
10. Настройте домен (опционально) или используйте IP адрес

ЧТО НУЖНО ПОДГОТОВИТЬ ПЕРЕД РАЗВЕРТЫВАНИЕМ
═══════════════════════════════════════════════════════════════════════════════

1. Убедитесь, что файл .env НЕ загружается в Git:
   - Создайте файл .gitignore в папке проекта
   - Добавьте в него строки:
     
     .env
     node_modules/
     analytics.db
     *.log
     
   - Это защитит ваш пароль от публикации

2. Проверьте package.json:
   - Убедитесь, что есть скрипт "start": "node server.js"
   - Это нужно для автоматического запуска

3. Обновите server.js (если нужно):
   - Убедитесь, что порт берётся из переменной окружения:
     const PORT = process.env.PORT || 3000;
   - Это важно для Render и других платформ

ШАГ 1: Подготовка проекта

ШАГ 4: Загрузка проекта на GitHub

   - Выберите "Открыть окно PowerShell здесь"
   
2. Выполните команды (по одной, нажимая Enter):
   
   git init

   git config --global user.email "ваш@email.com"
   git add .
   git commit -m "First commit"
   
3. На GitHub создайте новый репозиторий:
   - Зайдите на github.com

ЧАСТЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
═══════════════════════════════════════════════════════════════════════════════

ПРОБЛЕМА: "Build failed" при развертывании
РЕШЕНИЕ:
- Проверьте, что в package.json есть все зависимости
- Убедитесь, что Start Command: npm start
- Проверьте логи в Render (вкладка "Logs")

ПРОБЛЕМА: Сайт не открывается
РЕШЕНИЕ:
- Подождите 1-2 минуты (первый запуск медленный)
- Проверьте логи в Render
- Убедитесь, что переменные окружения добавлены

ПРОБЛЕМА: "Cannot find module"
РЕШЕНИЕ:
- Убедитесь, что все зависимости в package.json
- Проверьте, что Build Command: npm install

ПРОБЛЕМА: Пароль админки не работает
РЕШЕНИЕ:
- Проверьте переменную ADMIN_PASSWORD в Render
- Убедитесь, что нет лишних пробелов
- Перезапустите сервис (в Render нажмите "Manual Deploy")

ПРОБЛЕМА: Сайт "засыпает" на Render
РЕШЕНИЕ:
- Это нормально для бесплатного плана
- Первый запрос после простоя будет медленным (10-30 сек)
- Если нужно постоянно работать - используйте VPS (от 2-3$/мес)
