    КАК ВЫГРУЗИТЬ ОБНОВЛЕНИЕ НА GITHUB И RENDER
═══════════════════════════════════════════════════════════════════════════════

После оптимизации нужно загрузить изменения на GitHub.
Render автоматически обновит сайт!

═══════════════════════════════════════════════════════════════════════════════
ШАГ 1: ОТКРОЙТЕ POWERSHELL В ПАПКЕ ПРОЕКТА
═══════════════════════════════════════════════════════════════════════════════

1. Откройте папку: C:\Users\NewAdmin\Desktop\сайт
2. Зажмите SHIFT + правая кнопка мыши
3. Выберите "Открыть окно PowerShell здесь"

═══════════════════════════════════════════════════════════════════════════════
ШАГ 2: ПРОВЕРЬТЕ ИЗМЕНЕНИЯ
═══════════════════════════════════════════════════════════════════════════════

Выполните команду:

   git status

Вы должны увидеть измененные файлы:
   - index.html
   - server.js
   - ОТЧЕТ_ОПТИМИЗАЦИЯ.txt

═══════════════════════════════════════════════════════════════════════════════
ШАГ 3: ДОБАВЬТЕ ИЗМЕНЕНИЯ
═══════════════════════════════════════════════════════════════════════════════

Выполните:

   git add .

Или добавьте конкретные файлы:

   git add index.html server.js

═══════════════════════════════════════════════════════════════════════════════
ШАГ 4: СОЗДАЙТЕ КОММИТ
═══════════════════════════════════════════════════════════════════════════════

Выполните:

   git commit -m "Оптимизация производительности и мобильной версии"

Или более подробно:

   git commit -m "Оптимизация: улучшена производительность, оптимизирована мобильная версия, исправлены ошибки"

═══════════════════════════════════════════════════════════════════════════════
ШАГ 5: ОТПРАВЬТЕ НА GITHUB
═══════════════════════════════════════════════════════════════════════════════

Выполните:

   git push origin main

Если попросит авторизацию:
- В браузере откроется окно GitHub
- Нажмите "Authorize" (Разрешить)

═══════════════════════════════════════════════════════════════════════════════
ШАГ 6: ОЖИДАНИЕ ОБНОВЛЕНИЯ НА RENDER
═══════════════════════════════════════════════════════════════════════════════

1. Render автоматически обнаружит изменения (1-2 минуты)
2. Начнется автоматическое развертывание
3. Процесс займет 2-5 минут

Проверить статус:
1. Зайдите на https://render.com
2. Откройте ваш сервис (casino-landing)
3. Перейдите в раздел "Logs" (Логи)
4. Увидите процесс обновления

═══════════════════════════════════════════════════════════════════════════════
ПОЛНЫЙ СПИСОК КОМАНД (СКОПИРУЙТЕ И ВЫПОЛНИТЕ)
═══════════════════════════════════════════════════════════════════════════════

В PowerShell выполните по порядку:

   git status
   git add .
   git commit -m "Оптимизация производительности и мобильной версии"
   git push origin main

═══════════════════════════════════════════════════════════════════════════════
ЕСЛИ ВОЗНИКЛИ ПРОБЛЕМЫ
═══════════════════════════════════════════════════════════════════════════════

❌ "nothing to commit"
   РЕШЕНИЕ: Убедитесь, что вы в правильной папке (сайт)

❌ "fatal: not a git repository"
   РЕШЕНИЕ: Выполните git init (но обычно не нужно)

❌ "authentication failed"
   РЕШЕНИЕ: 
   1. В браузере разрешите доступ GitHub
   2. Или используйте Personal Access Token (см. инструкции выше)

❌ "push failed"
   РЕШЕНИЕ:
   1. Проверьте интернет-соединение
   2. Попробуйте снова: git push origin main

═══════════════════════════════════════════════════════════════════════════════
ПРОВЕРКА ОБНОВЛЕНИЯ
═══════════════════════════════════════════════════════════════════════════════

После успешного push:

1. Обновите страницу репозитория на GitHub:
   https://github.com/habibandrey18-create/casino-landing

2. Вы должны увидеть новый коммит с вашим сообщением

3. Подождите 2-5 минут

4. Откройте сайт:
   https://casino-landing.onrender.com

5. Проверьте, что изменения применились:
   - Сайт должен работать быстрее
   - На мобильных должно быть плавнее
   - Виртуальный помощник должен работать
